静态代理

角色分析:

  • 抽象角色: 一般会使用接口或抽象类来解决
  • 真实角色: 被代理的角色
  • 代理角色: 代理真实角色,一般会带有一些附属操作
  • 客户: 访问代理对象的人.

代理模式的好处:

  • 可以使得真实角色的操作更加纯粹,不用关注一些公共业务
  • 公共业务交给了代理角色,实现了业务的分工
  • 公共业务发生扩展时,方便集中管理.

代理模式的缺点: 代码量翻倍,开发效率变低.

  • 动态代理的角色和静态代理角色是一样的.
  • 动态代理的代理类是动态生成的,不是我们写好的.

动态代理分为两大类:

  • 基于接口 – JDK动态代理 我们使用这个
  • 基于类 – cglib

AOP的介绍

什么是AOP?

  • AOP为Aspect Oriented Programming的缩写。
    • 对OOP(Object Oriented Programming)的补充和延伸，
    • AOP 采用的是横向抽取机制,传统方法是纵向继承体系.

  • AOP 的思想:使用业务与服务代码分离

  • OOP 的思想:你做事,我放心

  • AOP 的好处:不加代码加功能

  • AOP 的底层原理是:动态代理

    • 静态与动态代理的区别:
      • 静态代理类是有程序员创建，在运行前，这些代理已经存在。
      • 动态代理在运行时，运用反射的机制创建

  • AOP 操作的相关术语

    1. 代理对象:需要增强的类
    2. 目标对象:增强后的代理类
    3. 连接点:可插入增强的方法
    4. 切点:插上了增强的方法
    5. 织入:把增强插入连接点的过程
    6. 增强(通知)advice:日志前,日志后,事务前,事务后
    7. 切面:事务,日志

AOP在Spring中的作用

提供声明式事务,允许用户自定义切面

  • 横切关注点: 跨越应用程序多个模块的方法或功能.
    即是,与我们业务逻辑无关的,但是我们需要关注的部分,就是横切关注点. 如 日志,安全,缓存,事务 等等 …
  • 切面(Aspect):横切关注点 被模块化 的特殊对象. 即,它是一个类。
  • 通知(Advice):切面必须要完成的工作.即,它是类中的一个方法。
  • 目标(Target): 被通知对象. --> 指真实角色Impl
  • 代理(Proxy): 向目标对象应用通知之后创建的对象。
  • 切入点(PointCut): 切面通知 执行的 "地点"的定义。
  • 连接点(JointPoint):与切入点匹配的执行点。
