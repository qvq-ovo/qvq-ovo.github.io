视图和逻辑分离

android设计时讲究的是逻辑和视图分离.因此不推荐在活动中直接编写界面的.
通用的做法是: 在布局文件中编写界面,然后在活动中引入.
就像这上面第二行就引用了setContentView(R.layout.activity_main);来调用了一个activity_main布局.

res目录中的资源分布

归纳一下: 1. 所有以`drawable`开头的文件夹都是用来放图片的. 2. 所有以`mipmap`开头的文件夹都是用来放应用图标的. 3. 所有以`values`开头的文件夹都是用来放字符串,样式,颜色等配置的. 4. `layout`文件夹是用来存放布局文件的.

引用资源的两种方法:
资源:

  • 在代码中使用R.string.hello_world获取该字段的引用
  • 在XML中通过@string/hello_world获取该字段的引用

string是可以替换的:
图片 — drawable
图标 — mipmap
布局文件 — layout

日志工具Log

Log类中一共提供了5中打印日志的方法.
Log.v() -> Log.d() -> Log.i() -> Log.w() -> Log.e().
级别是逐渐递增的.

以Log.d()为例:Log.d()要传入两个参数:tag , msg.tag用于打印信息的过滤.msg为具体的信息.
